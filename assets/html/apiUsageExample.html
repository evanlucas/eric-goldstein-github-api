GitHub API Server & Client Plugin

This API is RESTful and works via only a single route and HTTP verb:

GET /api/github/user/:repo_owner/repository/:repo_name
where :repo_owner corresponds to the repository owner's username and :repo_name corresponds to the name of the repository

Getting the Node.js Server Running

This Node.js application server relies on the use of an environment variable to notify the application of the OAuth token to use to connect to GitHub's API. Follow these steps to get it working:

  1. Login to your GitHub account on github.com
  2. Create a new "Personal Access Token" here: https://github.com/settings/tokens
  3. Copy your Personal Access Token string to your clipboard
  4. Clone the Git repository onto your local machine
  5. Navigate to the root of the cloned repository's directory
  6. Execute the following Bash command: GITHUB_TOKEN={personal_access_token} node server.js

API Usage Example

The GitHub repository used to store the code for this test is located here: https://github.com/mangs/eric-goldstein-github-api

By visiting that page, we know that the repository owner's username is mangs and the name of the repository is eric-goldstein-github-api. Thus, we'd fetch that repository's details via the following request:

GET /api/github/user/mangs/repository/eric-goldstein-github-api

The server should return JSON similar to the following:

{ "description": "Eric Goldstein's Github API Take Home Project.", "language": "JavaScript", "name": "eric-goldstein-github-api", "owner": "mangs", "url": "https://api.github.com/repos/mangs/eric-goldstein-github-api" }
